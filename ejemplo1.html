Secciones y líneas generales de un documento HTML5

La especificación HTML5 trae muchos nuevos elementos a los desarrolladores web permitiéndoles describir la estructura de un docuemnto web con semàntica estandarizada. Este documento describe esos elementos y como usarlos para definir el perfil de cualquier docuemnto.

Secciones y líneas generales de un documento HTML5

La definicion de la estructura de un documento en HTML4 y su algoritmo de perfilado es muy tosco y genera numerosos problemas.

1. HTML5 quita la necesidad de elementos < div> para definir secciones semànticas sin definir valores especifícos para los atributos class, introduciendo un nuevo elemento, < section>, el elemento de seccion HTML.

2. Mezclar varios documentos es difícil la inclusión de unsub-docuemtno en un documento principal. Estose resuelve en HTML5 con los elementos de seccionado (< article>, < section>, < nav>
y < aside>) son siempre subsecciones de su sección ancestre más cercana.

3. HTML5 introduce el elemento < hgroup> que oculta todos los elementos de cabecera excepto el primero de más alto rango (por ejemplo, < hgroup>< h1>Justine< /h1>< h2>Les Malheus de la Vertu< /h2>< /hgroup>crea el perfil 1.Justine).
